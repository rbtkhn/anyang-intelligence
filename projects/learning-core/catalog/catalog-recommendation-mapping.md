# Catalogue Recommendation Mapping

This note shows how the two Khan catalogue tracks should inform recommendations without collapsing into mastery claims.

The catalogue is a scaffold, not an authority layer.

It can expose available resource categories and candidate options. It cannot decide fit by itself. Every recommendation still needs the adapter layer, parent review, household constraints, and learner-use signals.

## Sample Mapping

### Case A: Parent wants a simple starter rhythm for a 7-year-old

Use:

- `khan_kids_curated_catalog`

Why:

- parent-supervised starter tool
- age-appropriate early-learning scope
- easier fit for a short daily block

Recommended framing:

- small supervised app block
- observe choice, stamina, frustration, and transfer
- connect strong signals into reading-basket or project follow-up
- treat the catalogue entry as an option source, not a reason to expand app use automatically

### Case B: Learner shows strong curiosity beyond the starter app

Use:

- `khan_academy_main_catalog`

Why:

- broader public course tree
- better for structured reinforcement or enrichment lookup
- better source for older siblings or advanced spillover interests

Recommended framing:

- optional extension pathway
- parent-approved, not assumed
- chosen for fit, not because the catalog itself proves readiness
- reviewed through household schedule, parent supervision, and learner-use evidence before entering the plan

## Hard Boundary

Neither catalogue track should be used to say:

- the student has mastered a subject
- the student belongs in a grade level
- the student has a diagnosis
- the student should override parent constraints
- the system should recommend a resource without parent-reviewed fit judgment
