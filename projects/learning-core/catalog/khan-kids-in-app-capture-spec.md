# Khan Kids In-App Capture Spec

This note defines the future governed intake path for richer Khan Academy Kids catalog coverage.

## Purpose

The public web does not appear to expose the full Khan Kids library as a clean external course catalogue.

Because of that, Elementary School should treat richer Khan Kids coverage as a **manual intake process**, not as an assumed public import surface.

## Intake Method

Operator records one entry or one batch at a time from the Khan Kids app library.

Each captured entry should include:

- stable id
- title shown in the app
- broad domain
- approximate level or age band if visible
- content type
- date captured
- capture note
- screenshot or operator note only if approved

## Evidence Rules

- Default evidence label: `manual-in-app-capture`
- Do not claim completeness for a section unless it was explicitly checked
- Do not infer hidden reading levels or hidden metadata not visible in the app
- Do not treat internal app sequencing as public doctrine unless it is actually observed

## Governance Boundary

In-app capture is for governed curation.

It is not permission to:

- scrape unofficially
- overclaim coverage
- save child-linked screenshots without approval
- convert app observations into mastery claims

## Recommended Batch Rhythm

- pick one domain or one book grouping
- capture visible entries
- label gaps honestly
- update the curated catalog only after review
