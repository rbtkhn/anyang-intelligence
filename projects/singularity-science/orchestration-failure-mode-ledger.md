# Orchestration Failure-Mode Ledger

This ledger records recurring ways a turn can look useful while still failing as a governed workflow.

Use it to tighten loops, templates, review gates, and training exemplars over time.

## Status Key

- `open`: recurring issue is visible but not yet addressed
- `tightening`: an artifact or loop revision is underway
- `watch`: partially improved; continue observing
- `resolved`: the failure mode has a durable mitigation surface
