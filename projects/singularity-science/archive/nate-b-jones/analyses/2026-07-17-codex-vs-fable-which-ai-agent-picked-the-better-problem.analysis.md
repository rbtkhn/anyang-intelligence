## Seam Analysis

### Seam: problem selection / automation-opportunity discovery

- Core claim: the real leverage may sit upstream, where an agent audits behavior and context to decide which problem is worth solving.
- Likely durable:
  - problem selection quality is a distinct value surface
  - a useful turn can include diagnosis plus artifact production
  - “pick the problem” is a meaningful evaluation pressure test
- Likely speculative:
  - the consistency with which a broad audit finds the highest-ROI problem
  - the degree to which this can be trusted in sensitive or noisy contexts
- Verification burden: medium
- Primitive opportunity: candidate for an automation-opportunity discovery gate that forces scope, exclusions, expected output shape, and approval stop points before a system audits a workspace
- Lane-safe routing opportunity: not yet; preserve first as an internal primitive candidate
- Membrane risk: medium-to-high; broad discovery can become an excuse to dissolve boundaries
- ROI disposition: primitive-candidate

### Seam: boundary-scoped context audit

- Core claim: open-ended discovery can be useful without total exposure if the operator can wall off private surfaces before the audit begins.
- Likely durable:
  - boundary-setting should happen before exploratory agent work
  - access scope is part of product design, not only security hygiene
  - “look broadly, but not everywhere” is a reusable operating pattern
- Likely speculative:
  - how often operators will set strong boundaries well
  - whether the resulting audits remain high-quality under tighter scope
- Verification burden: medium
- Primitive opportunity: route into permissions-and-authority review and scoped-discovery checklists
- Lane-safe routing opportunity: yes
- Membrane risk: low-to-medium if framed as a review question rather than a permission default
- ROI disposition: lane-test-ready

### Seam: comparative model portfolio strategy

- Core claim: ambiguous tasks may benefit from running more than one agent stack and selecting among their problem frames rather than trusting a single system.
- Likely durable:
  - diversity of judgment can be valuable when the task is unclear
  - comparative runs may reveal different bottlenecks and leverage points
- Likely speculative:
  - the cost-effectiveness of making this a routine operating pattern
  - the frequency with which comparison actually improves outcomes enough to justify the overhead
- Verification burden: high
- Primitive opportunity: reinforces useful-turn scoring and review-gate design, but not yet enough for a separate preserved primitive
- Lane-safe routing opportunity: not yet
- Membrane risk: medium; comparison workflows can look rigorous while multiplying supervision burden
- ROI disposition: needs-verification

- Verification burden: high

- Lane-safe routing opportunity: no

## Repeated-Seam Relevance

- What repeated from prior Nate B. Jones or adjacent Singularity Science sources:

  - operator trust is being shifted from raw model quality toward workflow structure
  - commentary rhetoric tends to outrun what one worked example can actually prove
- What looks new:

  - bounded workspace discovery as part of the prompt contract
  - comparative portfolio use as a way to harvest different strategic frames

  - more sources arguing that a useful turn begins with “pick the problem”
  - more sources treating access scoping as a first-class product feature

## Boundary

Do not generalize this source into a claim that exploratory agent audits should receive broad workspace access by default, or that one prompt can safely discover the most important problem in a lane. The durable takeaway is narrower: problem selection, scope boundaries, and harness ergonomics are all part of the turn product and deserve governed review.
