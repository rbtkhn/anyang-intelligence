# Learning Plan Skill

Use this skill to draft parent-reviewed Student Operating System learning plans.

This is a Student Operating System-level skill, not a generic lesson-planning skill.

## Purpose

Help Anyang Intelligence turn learner context into humane, parent-approved learning plans that:

- Fit the learner's interests, strengths, friction, and rhythm.
- Remain light enough to actually use.
- Preserve parent authority.
- Create portfolio-worthy evidence without over-documenting.
- Feed the next weekly parent review.

## Standard Output

When this skill is used, produce:

```text
Learning-plan checklist:
- Planning horizon:
- Parent goal:
- Learner profile signals:
- Activities:
- Evidence to save:
- Parent approval needed:
- Watch items:
- Review point:

Artifact:
<daily rhythm, weekly learning plan, activity plan, reading plan, project plan, or next-cycle plan>
```

## Procedure

### 1. Start From Profile And Parent Goal

Use:

- Learner profile.
- Parent goal.
- Student voice.
- Weekly parent review.
- Reading basket signals.
- Student portfolio evidence.
- Khan Academy Kids observations.
- Known friction or support signals.

If the learner profile is missing or stale, prepare a learner-profile update first.

### 2. Keep The Plan Humane

Prefer simple rhythms over crowded schedules.

A good plan includes:

- Reading.
- One focused practice area.
- One hands-on or creative activity.
- Movement, play, or outside time when appropriate.
- Parent observation.
- Evidence to save.

### 3. Preserve Parent Approval

Mark what the parent or guardian must approve before use:

- Daily rhythm.
- Curriculum or resource choice.
- Child-facing prompt.
- Sensitive support concern.
- Outside support.
- Any change that affects accountability requirements.

### 4. Close The Loop

Every plan should define:

- What evidence to save.
- What the parent should observe.
- When to review.
- What question the next weekly parent review should answer.

## Guardrails

Do not:

- Replace parent instruction.
- Over-automate the child's day.
- Diagnose.
- Grade.
- Treat app progress as proof of mastery.
- Ignore support concerns.
- Imply legal homeschool compliance.
- Use pricing or monetary-value language in parent-facing artifacts.

## Done When

The plan has a clear horizon, parent goal, learner-profile basis, humane activity set, evidence target, parent approval points, watch items, and review point.
