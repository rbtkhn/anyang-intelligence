# Student Experience Skill

Use this skill to design child-safe Student Operating System experiences without implying unsupervised direct system-student interaction.

This is a Student Operating System-level skill, not a generic education skill.

## Purpose

Help Anyang Intelligence prepare student-centered experiences that:

- Preserve parent mediation.
- Respect student agency.
- Use age-appropriate and low-pressure language.
- Save useful evidence without turning the child into a data source.
- Support learning without grading, diagnosing, or labeling.

## Standard Output

When this skill is used, produce:

```text
Student-experience checklist:
- Parent mediation:
- Age-appropriate tone:
- Low-pressure wording:
- Student agency:
- Evidence to save:
- Parent approval:
- Artifact to update:

Artifact:
<student survey, child reflection, proud-work prompt, learner-interest artifact, portfolio note, activity prompt, or experience plan>
```

## Procedure

### 1. Confirm Parent Mediation

Assume the parent or guardian mediates the experience unless explicitly approved otherwise.

State how the parent will:

- Introduce the prompt or activity.
- Observe the learner.
- Decide what to save.
- Approve any follow-up.

### 2. Keep The Student Experience Low-Pressure

Use wording that is:

- Short.
- Concrete.
- Kind.
- Optional where possible.
- Appropriate for the learner's age.
- Open to drawing, pointing, building, talking, moving, or choosing.

Avoid adult-operating language, performance pressure, and permanent labels.

### 3. Preserve Student Agency

Give the learner real but safe choice.

Useful prompts include:

- What are you proud of?
- What felt hard?
- What do you want to remember?
- What do you want to try next?
- Which piece should go in the student portfolio?
- Do you want to answer by talking, drawing, building, or showing?

### 4. Save Evidence Carefully

Choose the lightest useful evidence:

- Learner-interest artifact.
- Child reflection page.
- Proud-work note.
- Student portfolio entry.
- Parent observation note.
- Activity result.

Save only what helps the parent understand the child's learning, interest, pride, friction, or next support need.

### 5. Route Concerns To Parent Review

If the student signal suggests persistent reading, writing, attention, speech/language, math, behavior, health, or developmental concern, do not diagnose.

The next safe action is:

```text
Parent review and possible teacher, tutor, clinician, specialist, or other qualified human support.
```

## Guardrails

Do not:

- Diagnose.
- Grade.
- Label the child.
- Pressure the child for complete answers.
- Treat app progress as proof of mastery.
- Replace parent, teacher, tutor, clinician, or specialist judgment.
- Imply legal homeschool compliance.
- Use pricing or monetary-value language in student-facing artifacts.

## Done When

The student-centered work is parent-mediated, age-appropriate, low-pressure, agency-preserving, evidence-light, and tied to a clear artifact for parent review.
