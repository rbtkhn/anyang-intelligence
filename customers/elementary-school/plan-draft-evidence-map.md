# Plan Draft Evidence Map

This is an internal/operator worksheet for showing where each important part of a **30-Day Personalized Learning Plan** draft came from.

Use it after [onboarding-readiness-checklist.md](onboarding-readiness-checklist.md) and [plan-drafting-gate.md](plan-drafting-gate.md), but before a draft is sent for parent review.

The goal is simple:

- make learner-specific claims traceable
- keep assumptions visible
- keep unsupported sections open instead of invented

## Core Rule

If a learner-specific statement appears in the plan, the operator should be able to point to one of these sources:

- `Parent Input`
- `Parent-Approved Student Voice`
- `Approved Support Context`
- `Template Default`
- `Provisional Assumption`

If none of those apply, the line should not appear in the draft.

## Source Labels

Use these labels consistently:

- `Parent Input`
  - directly stated by the parent or guardian
- `Parent-Approved Student Voice`
  - child observation or quote included with parent approval
- `Approved Support Context`
  - outside-support context the parent explicitly allowed to be considered
- `Template Default`
  - safe non-specific structure from the plan template
- `Provisional Assumption`
  - a clearly labeled temporary assumption allowed by the readiness gate
- `Missing`
  - not known yet; leave open or request clarification

## Fast Checks

Before sending the draft, review this map and ask:

- Are any learner-specific claims sourced only from operator intuition?
- Are any `Template Default` rows pretending to be learner-specific?
- Are any `Provisional Assumption` rows unlabeled in the actual draft?
- Are any `Missing` rows being hidden by polished wording?
- Does any support-language row go beyond what the parent actually named?

If the answer to any of those is yes, revise the draft.

## Required Honesty Rules

- `Template Default` is allowed for structure, but not for fake specificity.
- `Provisional Assumption` is allowed only when the draft status is `Provisional`.
- `Missing` is better than invention.
- Evidence quality should drop to `thin` when important learner facts remain unknown.

## Parent Review Readiness

A draft is ready for parent review when:

- major learner-specific claims are traceable
- assumptions are visible
- missing inputs are visible
- approval-sensitive sections are clearly marked
- no diagnosis, grading, proof-of-mastery, pricing, or internal-skill language appears

## Completion Question

Before sending the draft, answer:

`If the parent asks "Why is this in the plan?" can I point to the source immediately?`

If not, revise the section before review.
