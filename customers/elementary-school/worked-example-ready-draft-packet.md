# Worked Example Ready Draft Packet

This is a fictional end-to-end example showing how the Elementary School lane should move from intake to a safe draft packet.

It uses the `Ready` scenario from [mock-intake-simulations.md](mock-intake-simulations.md). It is not a real family, student record, or delivery artifact.

Treat this as the canonical training fixture for a first safe 30-day plan draft. A future operator or agent should be able to read this packet and understand the minimum standard for moving from parent intake to a review-ready draft without inventing learner facts, weakening parent authority, or turning portfolio evidence into proof of mastery.

Use this packet as an operator example for:

- intake review
- readiness classification
- drafting gate use
- evidence-map discipline
- first safe plan draft
- parent approval handoff

## How To Use This Fixture

Use this packet when training or checking a new Elementary School drafting pass.

The operator should compare any future draft against this sequence:

1. Start from a parent intake signal.
2. Classify the intake as `Ready`, `Provisional`, or `Hold`.
3. Run the drafting gate before writing parent-facing plan language.
4. Map major plan claims back to parent inputs, approved context, template defaults, or labeled assumptions.
5. Draft only the sections that can be supported without diagnosis, grading, app-mastery claims, legal/accountability claims, or hidden child-facing authority.
6. Return the draft for parent approval before use.

This fixture is not a shortcut around intake. It is a pattern for how to preserve authority, evidence, and warmth while drafting.

## Minimum Passing Standard

A safe worked draft packet should:

- name the source intake scenario or parent-approved input
- state readiness classification and why drafting may proceed
- identify missing inputs or state that none block drafting
- keep the parent as the approval authority
- preserve privacy and saved/shared-information boundaries
- avoid diagnosis, grade-level claims, proof-of-mastery claims, and legal/accountability conclusions
- treat Khan Academy Kids as supervised practice and observation, not curriculum replacement or proof
- trace major draft sections back to evidence
- include parent approval questions before plan use

If a future packet cannot meet this standard, it should be marked `Provisional` or `Hold` rather than forced into a full draft.

## 1. Intake Summary

Source case: `Scenario 1: Structured New Homeschool Parent`

Key parent-approved inputs:

- First-month goal: create a calm learning rhythm and help the child feel proud of daily work.
- Plan avoid: too many worksheets, pressure, long app blocks.
- Approver: parent approves the plan before use.
- Privacy: save parent notes, portfolio photos, and proud work; do not share outside the household without parent approval.
- Accountability: parent is handling homeschool requirements separately; the plan supports daily learning only.
- Learner: 7-year-old, early elementary.
- Rhythm: mid-morning, Monday through Thursday.
- Parent time: 45 minutes on light days, 90 minutes on fuller days.
- Reading comfort: enjoys read-alouds and easy readers, resists long independent reading.
- Math comfort: likes counting games and dice, gets frustrated with written pages.
- Writing/drawing comfort: loves drawing, writes short labels when prompted.
- Interests: animals, rocks, nature walks, drawing, building with blocks.
- Proud of: a drawing notebook and identifying birds outside.
- Friction: avoids long reading and math worksheets.
- Learning modes: drawing, building, moving, listening.
- Resources: books at home, paper, pencils, art supplies, dice, playing cards, outdoor space.
- Khan Academy Kids: installed and approved for 15 minutes on learning days, parent-supervised.
- Reading basket: several animal books, easy readers, one parent read-aloud available.
- Portfolio: physical folder plus parent phone photos; parent decides what is saved; child can choose proud work weekly.
- Watch items: reading avoidance and worksheet frustration; no high-stakes concern named.
- Approval before use: parent wants to approve daily rhythm, reading basket, app rule, portfolio plan, and weekly review questions.

## 2. Readiness Classification

- Status: `Ready`
- Reason: authority, privacy, learner context, schedule, resources, app decision, reading basket status, and portfolio approach are all sufficiently clear to draft without invention.
- Missing inputs: none that block a draft.
- Assumptions: none required for status.

## 3. Drafting Gate Result

The draft may proceed because:

- the case is `Ready`
- parent approval authority is clear
- privacy and sharing boundaries are clear
- the plan can be built from actual parent inputs
- learner-specific claims do not require diagnosis, placement claims, or unsupported labels

Guardrails still active:

- do not assign exact reading or math levels
- do not present app use as mastery
- do not imply legal or accountability compliance
- do not exceed the parent-approved app rule

## 5. Example Draft

Below is a safe worked example of the parent-facing plan shape.

```text
30-Day Personalized Learning Plan:
Plan status: Ready
Learner: Parent-approved learner label
Parent goal: Create a calm learning rhythm and help the child feel proud of daily work.
Inputs checked: Parent goal, approval authority, privacy rules, learner context, schedule, resources, Khan Academy Kids decision, reading basket status, portfolio setup, watch items.
Missing inputs: None blocking draft.

Authority and safety boundary:
- The parent approves this plan before use.
- The parent decides what is saved, what is shared, and what changes.
- This plan supports daily learning but does not diagnose, grade, prove mastery, or satisfy legal/accountability requirements by itself.

New-student onboarding sequence:
Week 0 / Before start:
- Confirm the mid-morning Monday-Thursday rhythm.
- Confirm the 15-minute Khan Academy Kids rule.
- Pull together animal books, easy readers, and the parent read-aloud.
- Set up the physical folder and phone-photo portfolio routine.
- Choose a weekly parent review time.
Week 1 setup:
- Keep sessions short enough to create early wins.
- Watch what the child chooses, avoids, and feels proud of.
- Save only light useful evidence.

Learner context summary:
- Interests: animals, rocks, nature walks, drawing, building with blocks.
- Strengths: enjoys drawing, takes pride in a drawing notebook, notices birds outside.
- Friction or support signals: resists long independent reading and worksheet-heavy math.
- Parent constraints: learning works best mid-morning Monday-Thursday; about 45 minutes on light days and up to 90 minutes on fuller days.
- Evidence quality: medium; enough to draft safely, still early in the learning arc.

30-day rhythm:
- Daily anchor: calm mid-morning learning block, Monday-Thursday.
- Reading: short read-aloud plus easy access to confidence books.
- Khan Academy Kids: 15 minutes, parent-supervised, used for practice and observation only.
- Hands-on practice: counting games, dice, cards, drawing labels, simple making activities.
- Project / enrichment: nature, animals, rocks, drawing, building.
- Movement / play / outside time: built into the routine each learning day.
- Parent observation: notice pride, avoidance, repeated interests, and frustration points.
- Portfolio evidence: save only proud work, meaningful photos, and short parent notes.

Week 1 focus:
- Goal: settle into a calm, low-pressure rhythm.
- Activities:
  - animal-themed read-aloud and easy book browsing
  - short Khan Academy Kids support block
  - drawing or nature observation notebook
- Evidence to save:
  - one proud drawing
  - one parent observation note
  - one photo or notebook page if useful
- Parent review question:
  - What helped the child settle into the rhythm, and what felt like too much?

Week 2 focus:
- Goal: increase confidence through easy wins and hands-on math.
- Activities:
  - dice or card counting games
  - easy-reader time with parent support
  - building or sorting activity tied to interests
- Evidence to save:
  - one math game note
  - one reading choice note
  - one proud work sample
- Parent review question:
  - What should we repeat, pause, or make easier next week?

Week 3 focus:
- Goal: deepen interest-led engagement through nature, drawing, and making.
- Activities:
  - short nature walk or outdoor observation
  - drawing notebook entry
  - simple build or create activity
- Evidence to save:
  - one drawing or photo
  - one question the child asks
  - one note about repeated interests
- Parent review question:
  - What pattern is becoming visible in the child's interests, friction, or pride?

Week 4 focus:
- Goal: review what worked and prepare the next month.
- Activities:
  - favorite-book revisit
  - favorite math or counting game revisit
  - choose proud work for the portfolio
- Evidence to save:
  - 1-3 pieces the parent wants to keep
  - one short note about what helped most
- Parent review question:
  - What did this month teach us about the next best month?

Reading basket plan:
- Confidence books: easy readers already available at home.
- Current-level books: parent-selected easy reading options.
- Stretch book: one parent-supported book.
- Interest-led nonfiction: animal or nature book.
- Parent read-aloud: current shared read-aloud already available.
- Optional fun format: nature or animal picture-rich material if desired.

Khan Academy Kids use rule:
- Use for 15 minutes on learning days.
- Parent-supervised.
- Treat as practice and observation, not as a grade, diagnosis, curriculum replacement, or proof of mastery.

Portfolio evidence plan:
- Use the physical folder plus phone photos.
- Save proud work, useful photos, and short notes only when they help reveal the learning arc.
- Let the child help choose proud work weekly when it feels easy.

Weekly parent review prompts:
- What did the child enjoy?
- What did the child avoid?
- What felt too easy?
- What felt too hard?
- What should we repeat next week?
- What should we pause?

First monthly portfolio review shape:
- This month we noticed:
- Proud work:
- Milestones:
- Growth evidence:
- Recurring interests:
- Friction or support needs:
- Recommended next steps:
- Portfolio pieces to keep:
- Parent decision needed:

Parent decisions needed:
- Does the daily rhythm still feel realistic?
- Does the reading basket feel right?
- Does the 15-minute app rule still feel good?
- What should be repeated next month?

Watch items / support signals:
- Reading avoidance during longer independent reading.
- Frustration with worksheet-heavy math.
- Any shift in pride, curiosity, or distress that the parent wants reviewed.

Next-month decision options:
- Continue the rhythm with small adjustments.
- Refresh the reading basket.
- Add more hands-on math if current games are working.
- Deepen nature/drawing/project work.
- Pause and simplify if the current rhythm feels too full.
```

- Khan Academy Kids rule

- portfolio evidence plan
- week 1 focus
- what information may be saved
- what information may be shared

## 7. Why This Example Matters

This packet demonstrates that the lane can now:

- classify intake safely
- draft from actual parent-approved inputs
- avoid invention
- trace plan claims back to evidence
- preserve parent authority before use

That is the current target state for a safe first-month plan draft in Elementary School.

## 8. Agent Training Notes

When using this fixture to train a future agent, the desired behavior is not merely to copy the example plan. The desired behavior is to preserve the chain of authority:

```text
parent intake
  -> readiness classification
  -> drafting gate
  -> evidence map
  -> bounded draft
  -> parent approval
  -> learning update after review
```

Strong future drafts should sound warm and useful, but the safety test is structural:

- no unsupported personalization
- no invented resources
- no hidden curriculum decisions
- no claim that app activity proves mastery
- no private information reuse without approval
- no child-facing delivery before parent approval

The reusable primitive here is the disciplined handoff from evidence to draft. The specific child details in this fixture are fictional and should not be reused as if they describe a real learner.
