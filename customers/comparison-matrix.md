# Customer Comparison Matrix

This matrix compares current customer installations so Anyang Intelligence can see what is common across domains and what should remain customer-specific.

## Portfolio View

| Customer | Domain | Cadence | Primary OS job | Trust level |
| --- | --- | --- | --- | --- |
| [Book Club](book-club/README.md) | Recreational community | Monthly | Keep selection, hosting, discussion, and memory easy to sustain | Low |
| [Elementary School](elementary-school/README.md) | K-5 educational operations | Daily / weekly / school-year cycles | Coordinate student support, staff workflows, family communication, and safety-aware follow-up | Very high |
| [Grace Gems](grace-gems/README.md) | Marketplace business | Daily / weekly | Gate listings, protect margin, improve marketplace execution, and learn from customer signals | Medium |
| [Media Production](media-production/README.md) | Digital media production | Weekly / campaign-based | Move ideas through editorial judgment, production, quality gates, distribution, and audience learning | Medium |
| [Mountain Home](mountain-home/README.md) | Property and asset governance | Weekly / monthly / seasonal | Rank risk, preserve continuity, coordinate maintenance, and prepare for seasonal hazards | High |
| [Non-Profit](non-profit/README.md) | Mission-driven organization | Monthly / board / grant cycles | Align mission, programs, donors, volunteers, reporting, and impact evidence | High |

## Software Primitives Implied

| Primitive | Customers that need it | Product implication |
| --- | --- | --- |
| Context map | All | Every install starts with a structured operating model |
| Memory object registry | All | The product needs configurable domain objects |
| Decision log | All | Decisions should be durable, reviewable, and linked to evidence |
| Operating review | All | Reviews are the heartbeat of the OS |
| Risk register | Elementary School, Grace Gems, Mountain Home, Non-Profit, Media Production | Risk exists across domains, but severity and governance vary |
| Calendar / cadence engine | All | The OS needs recurring reviews, due dates, seasonal gates, and event triggers |
| Quality gate | Grace Gems, Media Production, Elementary School, Non-Profit | Gates prevent premature external action |
| Communication draft queue | Elementary School, Non-Profit, Grace Gems, Book Club, Media Production | AI may draft, but humans approve external messages |
| Evidence / proof ledger | Grace Gems, Mountain Home, Non-Profit, Media Production, Elementary School | Claims, decisions, and reviews need receipts |
| Learning loop | All | Each install should update future decisions from outcomes |

## Emerging Product Thesis

Anyang Intelligence is one product if it can keep the primitives stable while letting each customer configure:

- Memory objects.
- Review cadence.
- Decision types.
- Governance strictness.
- Quality gates.
- Evidence standards.

The customer-specific layer should describe the domain. The shared product layer should provide the operating spine.
